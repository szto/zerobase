# zerobase 제품 방향

> 0에서 시작해 자신만의 비즈니스와 세계를 구축하는 플랫폼.

- **타깃**: 스몰 비즈니스 / 크리에이터
- **첫 형태**: 비즈니스 사이트 (소개 + 서비스 + 문의 수집)
- **원자 단위 "월드"**: URL(`/w/<slug>/`) + 소유자 + 블록들 + 수집 데이터(문의)

## 설계 원칙

1. **블록 조립, 자유 생성 금지** — 만들어진 것은 반드시 돌아간다.
   블록 스키마의 단일 소스는 `app/blocks.py` 의 `BLOCK_TYPES`.
2. **공개 페이지는 가볍게** — `/w/<slug>/` 는 일반 서버 렌더 (WS 없음).
   LiveView 는 스튜디오(편집)에만 사용한다.
3. **즉시 저장** — 에디터에 저장 버튼이 없다. 모든 편집은 이벤트 즉시 DB 반영,
   미리보기는 djust 가 실시간 갱신.

## 로드맵

- [x] **Phase 1 — 손으로 만드는 월드**: 가입 → 월드 생성(시작 블록 4종) →
      실시간 에디터 → 공개 페이지 → 문의 수집
- [ ] **Phase 2 — 대화로 만드는 월드**: AI(Claude API)가 사용자의 설명을
      블록 조립으로 번역. AI 는 코드를 쓰지 않고 블록만 조작한다.
- [ ] **Phase 3 — 승격과 수익화**: 커스텀 도메인, 전용 서비스 승격
      (템플릿+Coolify 팩토리 재사용), 결제 블록.

## 블록 추가 방법

`app/blocks.py` 의 주석 참고 — 레지스트리 + 파셜 + Type choices 세 곳.
